Learning About Service Workers and PWA's

PWA stands for Progressive Web Applications and it is a type of app that can be used when the user is offline and the user can also install it in their device. PWA's are fast, reliable and engaging. Advantages of building PWA's according to the lecture are:

  • Worthy being on the home screen - when the PWA criteria is met, Chrome will prompt users to add the PWA to their home screen
  • Reliability no matter the network status - service workers help sending data faster which will then allow users to complete tasks faster
  • More engagement - depending on how you use the push notifications, you might increase engagement which might directly or indirectly increase sales

In order for a PWA app to work it needs a basic HTML file, Javascript file, Manifest file and a Service-Worker file is a must, and a CSS for styling purposes. But remember that service-worker will only work with https otherwise it won't work. The way the service worker file works is by first registering (if the browser supports it. If the browser does support then the registering process beigns with a promise method. The promise method installs the service worker, then it activates it with an event (by storing all the files) and then it fetches it (which is responsable for fetching all the cashed files in case the user is offline).
